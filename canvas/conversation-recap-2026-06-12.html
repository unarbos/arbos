Diagram unpacked — Per-Composer Adapters on a Shared Surrogate

Distribute one shared surrogate of an expert; let each consumer attach a tiny residual adapter that corrects it for their own routed-token distribution, without replacing the authoritative expert.

  • Owner Composer A holds the real expert E ᵢ and fits a canonical surrogate S ᵢ from it — A owns the authoritative meaning.
  • S ᵢ is distributed once to all remote composers.
  • Composers B, C run base S ᵢ + own adapter; local view S ᵢ, X(x) = S ᵢ(x) + A ᵢ, X(x).
  • Constraint: adapters are trained/approved against the real E ᵢ, not allowed to drift on local loss alone.
  • Win: one object captures the global expert; each consumer pays only for a tiny residual; canonical expert stays single source of truth.

Reads as a mixture-of-experts / federated framing — "expert i", "routed-token distribution".
