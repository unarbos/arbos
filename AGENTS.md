# AGENTS.md

## Cursor Cloud specific instructions

### Overview

Arbos is a Discord bot that orchestrates "RALPH loops" — autonomous AI agent execution cycles using the `claude` CLI. It's a single Node.js/TypeScript service managed by PM2.

### Prerequisites (already installed in the VM)

- Node.js v22+ and npm
- PM2 (global): `npm install -g pm2`

### Key commands

| Task | Command |
|------|---------|
| Install deps | `npm install` |
| Build | `npm run build` |
| Type-check / lint | `npx tsc --noEmit` |
| Dev mode | `npm run dev` (builds, starts PM2, runs tsc --watch) |
| Start via PM2 | `pm2 start ecosystem.config.cjs` |
| View logs | `pm2 logs arbos` |

### Secrets required to run

The app needs three secrets, stored in an AES-256-GCM encrypted vault at `~/.arbos/vault.enc`. They can also be provided as environment variables:

- `DISCORD_TOKEN` — Discord bot token
- `GUILD_ID` — Discord server (guild) ID
- `OPENROUTER_KEY` — OpenRouter API key for Claude access

The vault is unlocked via `VAULT_KEY` in `.env`. A `.env` file with `VAULT_KEY` and `WORKSPACE_ROOT` is generated by `run.sh` or can be created manually.

### Gotchas

- The app uses `--env-file=.env` (Node.js native flag) instead of just `dotenv`, so PM2 passes it via `node_args` in `ecosystem.config.cjs`.
- There is no ESLint config — TypeScript strict mode (`npx tsc --noEmit`) is the lint check.
- There are no automated tests in this repository.
- The `npm run dev` script chains `tsc && pm2 start ecosystem.config.cjs && tsc --watch` — it requires valid credentials to be useful since PM2 will start the bot.
- PM2 watches the `dist/` directory for changes and auto-restarts on recompile.
